📝 플레이데이터 주간회고 (WIL)

📆 2025년 7월 15일 ~ 2025년 7월 21일 (3주차)

🥅 주간회고의 목적

지난 일주일을 되돌아보며 현재 상태를 파악하고, 더 나은 다음 주를 위해 어떤 부분을 어떻게 채워나갈지 고민하기 위함.

💡 추천하는 템플릿: KPT (Keep, Problem, Try)

Keep (만족하고 계속 이어갈 부분)

  • 데이터 분석 기초 다지기: 데이터 분석의 전반적인 프로세스와 핵심 도구(Numpy, Pandas)에 대한 이론적 이해를 탄탄히 다질 수 있었습니다. 특히 Pandas를 활용한 데이터 핸들링에 대한 개념을 잘 잡을 수 있었습니다.
  • 시각화 도구 학습: Matplotlib과 Seaborn을 통해 다양한 그래프를 그리고 데이터를 시각화하는 방법을 익혔습니다. 기본적인 그래프 종류와 각 라이브러리의 특징을 이해하는 데 집중했습니다.
  • 미니 프로젝트 주제 선정 및 구체화: 조원들과 협력하여 ‘24절기 변화 예측’이라는 흥미로운 주제를 선정하고, 데이터 수집 및 분석 방향에 대해 깊이 있는 논의를 진행하며 구체적인 계획을 세울 수 있었습니다.

Problem (개선이 필요하다고 생각되는 부분)

  • 데이터 전처리 실습 부족: 변수 변환, 결측치/이상값 처리, 데이터 정규화 등 데이터 전처리 과정의 중요성은 인지했으나, 실제 대규모 데이터셋에 적용해보는 실습 경험이 부족했습니다. 이론과 실습 간의 괴리가 느껴집니다.
  • 라이브러리 숙련도: Numpy의 배열 연산, 인덱싱, 슬라이싱, Pandas의 데이터프레임 조작 등 기본적인 기능은 이해했지만, 실제 데이터를 다룰 때 어떤 함수를 어떻게 적용해야 할지 바로 떠오르지 않는 경우가 많았습니다. 숙련도를 높일 필요성을 느낍니다.
  • 미니 프로젝트 데이터 기준 모호성: ‘24절기와 날씨 데이터 분석’ 프로젝트에서 ‘실제 절기’를 정의하는 기준이 초반에 다소 모호하여, 어떤 데이터를 어떻게 가공해야 할지 혼란스러웠습니다. 이 부분에서 시간 소요가 있었습니다.

Try (Problem에 대한 해결책, 실행 가능한 것)

  • 데이터 전처리 집중 실습: 다음 주에는 다양한 형태의 실제 데이터셋을 직접 불러와 결측치 처리, 이상값 탐지 및 처리, 데이터 타입 변환 등의 전처리 과정을 집중적으로 실습할 예정입니다. 특히, 미니 프로젝트에서 사용할 기상 데이터에 바로 적용해볼 것입니다.
  • Numpy/Pandas 코드 연습: 매일 일정 시간 동안 LeetCode나 프로그래머스 등의 플랫폼에서 Numpy와 Pandas를 활용한 데이터 조작 문제를 풀어보며 코드 작성 속도와 문제 해결 능력을 향상시킬 것입니다.
  • 미니 프로젝트 기준 명확화 및 적용: 오늘 논의했던 ‘실제 절기’ 정의 기준(예: 일평균 기온 0℃ 이상 5일 지속 = 실제 입춘)을 바탕으로 기상 데이터를 분석하고, 각 절기별 날짜 차이를 계산하는 코드를 작성할 예정입니다. 이 기준을 적용하여 데이터 크롤링 및 전처리를 진행할 것입니다.

📜 트러블 슈팅 (이번 주 겪은 문제와 해결 과정)

문제 1: 주피터 노트북 환경 충돌 및 접근 거부 에러

  • 개요: 특정 라이브러리 설치 후 주피터 노트북 실행 시, access denied 메시지와 함께 정상적으로 작동하지 않는 문제가 발생했습니다.
  • 해결책: 기존에 열려 있던 모든 주피터 노트북 창과 아나콘다 프롬프트를 완전히 종료하고, 재부팅 후 다시 실행하니 문제가 해결되었습니다. (system) 추후 유사 문제 발생 시 가장 먼저 시도할 조치임을 인지했습니다.

문제 2: Plotly app.server.run() 경고 메시지 및 로컬 접속 문제

  • 개요: Plotly를 이용한 대시보드 개발 중 Environment: production WARNING: This is a development server. Do not use it in a production deployment. 와 같은 경고 메시지가 뜨며, 로컬 환경에서 웹 페이지에 접속이 원활하지 않았습니다.
  • 수정 전: if __name__ == '__main__': app.server.run()
  • 수정 후 (others): if __name__ == '__main__': app.server.run(host='0.0.0.0')
    • host='0.0.0.0'을 추가하여 개발 환경과 동일하게 IP 주소를 지정하니, 포트를 5000번으로 자동으로 지정해주어 문제없이 접속되었습니다.
  • 추가 문제: host='0.0.0.0'으로 설정 후, 포트 5000번이 이미 사용 중이라는 Address already in use 에러가 발생하는 경우가 있었습니다. 이는 이전에 실행했던 서버 프로세스가 종료되지 않아 발생하는 문제로 파악했습니다. 해당 포트를 사용 중인 프로세스를 강제 종료하거나 다른 포트 번호를 지정하여 해결해야 함을 알게 되었습니다.
