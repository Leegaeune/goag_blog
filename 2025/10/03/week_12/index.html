🔑 이번 주에 배운 중요한 개념

1. Output Parsing

모델 응답을 원하는 형태로 변환해야만 후처리 로직을 단순화할 수 있다.
특히 JSON 파싱은 추천, 검색, 요약 등 실용적인 서비스에 필수.

2. Conditional Chain

조건에 따라 다른 체인을 실행하게 하면 분기 처리가 가능하다.
→ 예: 긍정적 피드백이면 요약, 부정적 피드백이면 개선안 제안.

3. Memory

대화형 시스템의 핵심은 “기억”.
히스토리를 저장하고 불러오는 구조를 직접 구현해 보면서,
단순한 API 호출을 넘어 대화 지속성을 구현할 수 있었다.

Keep / Problem / Try

Keep

  • 체인 구조를 직접 설계해보는 습관
  • Parser를 적극 활용해 모델 응답을 구조화
  • 작은 단위부터 통합하는 방식 유지

Problem

  • 체인이 복잡해질수록 입력/출력 형식이 혼란스러움
  • Memory 세션 관리가 꼬이면 대화 기록이 뒤섞임
  • JSON 응답이 가끔 깨져서 예외 처리 필요

Try

  • Chain 실행 시 assert로 입력/출력 형식 검증
  • 세션 관리 유틸 함수를 따로 만들어 모듈화
  • JSON 파싱 실패 시 예외 복구 로직 추가

🌟 마무리 소감

이번 주는 LLM과 LangChain의 실전 활용에 본격적으로 들어간 한 주였다.
특히 Chroma DB, Chain, Memory는 앞으로 내가 만들 프로젝트(문서 기반 Q&A 시스템)의 뼈대를 구성할 핵심 기술임을 알게 되었다.

지난주가 딥러닝의 “모델 진화 과정”을 이해한 시간이었다면,
이번 주는 그 모델을 실제 서비스로 연결하는 방법을 배운 시간이었다.

실험 과정에서 수많은 오류와 시행착오가 있었지만,
이 과정을 통해 “AI 모델을 서비스로 구현한다”는 게 단순한 API 호출이 아니라
데이터 저장 → 체인 설계 → 응답 파싱 → 히스토리 관리까지 아우르는 종합적인 작업임을 체감했다.

앞으로 진행할 내외부 문서 기반 질의응답 시스템 프로젝트에서
이번 주에 배운 기술들을 실제로 활용할 수 있을 것 같아 기대된다.
