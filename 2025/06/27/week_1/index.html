📝 플레이데이터 주간회고



📌 SK네트웍스 Family AI 캠프를 시작하기 앞서,,

AI융합학과가 본전공이고 컴퓨터공학은 다전공이지만, 실제 AI나 데이터 코딩 역량은 아직 부족하다고 느꼈다.

단순 이론이나 개념 공부만으로는 실전에서 무엇을 만들고 적용하는 자신감이 생기지 않았기에 내 손으로 직접 모델을 만들고, 데이터를 다루며 의미 있는 결과를 내는 경험을 위해 캠프에 참가하게 됐다.


📚 이번 주 학습 내용 (Facts)

  • 자료형

    • 숫자형, 불리언, 문자열, 리스트/튜플, 세트, 딕셔너리 기본 문법 학습
    • len(), type(), sum() 같은 내장 함수 활용

  • 조건문 & 반복문

    • if-elif-else 문으로 조건 분기 처리
    • for 반복문으로 리스트/문자열 순회
    • while 반복문으로 조건 제어

  • 함수

    • 매개변수와 반환값의 개념 이해
    • 중복되는 코드를 함수로 묶어 재사용성 확보
    • lambda 간단 활용

  • 브랜치 (조건 중첩)

    • 여러 조건을 단계적으로 나누어 처리하는 구조 연습
    • 복잡한 문제를 작은 단위로 쪼개는 습관 강화

🛠️ 문제 & 해결 과정 (Troubleshooting Log)

1) IndexError: list index out of range

  • -> 리스트 길이보다 큰 인덱스 접근
  • 해결: len()으로 길이 확인 후 for value in list: 형태로 순회
  • 체크: 범위 밖 인덱스 접근은 무조건 에러. 컬렉션 자체 순회가 더 안전

2) SyntaxError & 들여쓰기

  • -> if문 블록 들여쓰기 불일치
  • 해결: 코드 포맷터/자동정렬 사용
  • 체크: 파이썬에서 들여쓰기는 문법 그 자체

Keep

  • 에러 메시지를 끝까지 읽고 해석하는 습관

  • print()/로그로 중간값 확인하며 원인 범위 축소

  • 반복되는 코드를 함수화하여 가독성 유지

Problem

  • 오타/들여쓰기 같은 기초 실수로 시간 소모

  • 설계 없이 바로 코딩 → 흐름 복잡/가독성 저하

  • 중첩 조건/반복에서 로직 추적 어려움

Try

  • 에러별 노트 구축(IndexError, TypeError, KeyError 패턴/원인/해결 템플릿)

  • “주석으로 로직 스케치 → 코드 작성 → 실행/디버깅 → 리팩토링” 루틴화

  • 입출력/엣지케이스(빈 리스트, 경계 인덱스, 누락 키) 먼저 정의


🧑‍💻 1주차: 파이썬을 다시 바라보다

이번 주에는 파이썬 기초부터 함수 사용법까지 다양한 내용을 직접 코딩하며 감각을 익혔다.
파이썬은 문법이 간단하지만, 사소한 실수가 프로그램 전체를 멈추게 한다는 점을 체감했다.
반복문과 조건문을 결합해 문제를 단계적으로 분해하는 습관이 중요 !!
