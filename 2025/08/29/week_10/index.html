📝 플레이데이터 주간회고



📚 이번 주 학습 내용 (Facts)

  • RNN (Recurrent Neural Network)

    • 순차 데이터 처리의 기본 구조
    • 시간에 따라 hidden state가 어떻게 전달되는지 직접 구현
    • Gradient Vanishing 문제를 실습으로 체감

  • LSTM / GRU

    • RNN의 한계를 보완한 구조
    • LSTM: Forget/Input/Output 게이트가 정보를 선택적으로 업데이트
    • GRU: 게이트를 단순화해 효율적이면서도 성능 유지

  • 텍스트 분류

    • nn.Embedding을 활용한 단어 벡터 표현 학습
    • 토큰화 및 vocab 구축 과정에서 작은 실수가 전체 모델에 영향 준다는 걸 경험

  • Seq2Seq

    • 인코더-디코더 구조로 번역 모델 기초 구현
    • Teacher Forcing 기법으로 학습 안정화 확인

  • Attention

    • Query, Key, Value 구조를 통해 문맥을 동적으로 반영
    • Seq2Seq에 Attention을 붙였을 때 번역 성능 향상 확인
    • 단순히 마지막 hidden state에 의존하지 않고 모든 시점의 정보를 활용할 수 있다는 점이 핵심

  • 전이학습 (Transfer Learning)

    • 사전 학습된 모델을 불러와서 Feature Extractor 혹은 Fine-tuning 방식으로 재활용
    • 학습 시간을 크게 단축할 수 있음을 실감

🧑‍💻 깨달은 점 (Insights)

  • Shape 확인 습관화
    에러보다는, 매번 print(tensor.shape)로 흐름을 따라가니 Attention 구조가 머릿속에 그려졌다.

  • 추상적 개념이 코드로 구체화된다
    논문 그림으로만 보던 Attention이 실제 텐서 연산으로 구현되면서 “진짜로 돌아가는 원리”가 이해됐다.

  • 작은 실수가 큰 혼란을 만든다
    Vocab 크기 하나 잘못 지정했을 뿐인데 모델 전체가 안 돌아갔다.
    데이터 전처리의 중요성을 다시 느꼈다.

  • 딥러닝의 발전사는 문제 해결의 연속
    RNN → LSTM/GRU → Attention → Transformer로 이어지는 흐름이 단순한 기술 업그레이드가 아니라, 실제 한계를 해결해 나가는 과정임을 알게 됐다.

🔑 이번 주에 배운 중요한 개념

1. RNN의 한계와 LSTM의 필요성

RNN은 과거 정보를 전달할 수 있지만, 문장이 길어질수록 정보가 희미해지는 Vanishing Gradient 문제가 심각했다.
직접 긴 시퀀스를 학습시키다 보니 초반 단어의 의미가 거의 반영되지 않는 걸 확인할 수 있었다.
이때 LSTM의 게이트 구조가 왜 등장했는지 명확히 이해할 수 있었다.

python

간단한 LSTM 예시

lstm = nn.LSTM(input_size=128, hidden_size=256, batch_first=True)
out, (h, c) = lstm(inputs) # h: hidden state, c: cell state


2. Seq2Seq와 Attention의 결합

Seq2Seq는 인코더가 입력 전체를 하나의 벡터로 요약하는데, 긴 문장에서 정보 손실이 컸다.
Attention을 추가하자, 디코더가 각 시점마다 입력의 다른 부분에 집중할 수 있게 되었다.
직접 구현한 코드에서 softmax 가중치가 달라지는 걸 보니, “모델이 정말 단어마다 집중하는구나” 실감이 났다.

#Attention 핵심 부분
scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)
attn_weights = F.softmax(scores, dim=-1)
context = torch.matmul(attn_weights, value)


3. 전이학습의 효율성

처음부터 모델을 학습시키면 시간이 오래 걸렸는데,
사전 학습된 모델을 불러와 Linear Layer만 얹어 fine-tuning 하니 훨씬 빠르게 결과가 나왔다.
현업에서는 왜 Transfer Learning을 많이 쓰는지 알게 되었다.

#전이학습 예시
model = models.resnet18(pretrained=True)
for param in model.parameters():
param.requires_grad = False # Feature extractor로 사용
model.fc = nn.Linear(model.fc.in_features, num_classes) # 새로운 분류 레이어

🌟 마무리 소감

이번 주는 딥러닝 모델의 “진화 과정”을 따라간 기분이었다.
RNN의 한계를 직접 느끼고, LSTM/GRU로 개선하는 방법을 배우고,
마지막에 Attention으로 문맥을 더 잘 반영하는 모델을 구현해 보니
자연스럽게 Transformer로 이어질 준비가 된 것 같다.

단순히 이론으로 배우는 것보다, 코드에서 발생하는 차원 오류와 디버깅 과정이
오히려 모델 구조를 더 깊이 이해하게 만들어 주었다.
앞으로 BERT, GPT 같은 모델을 배울 때도 이번 경험이 큰 기반이 될 것 같다.
